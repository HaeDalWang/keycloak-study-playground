# 챕터 8: Keycloak as Code

> 가상 시나리오: 설정 실수로 Realm 망가짐 → 코드로 관리해서 복원/배포 자동화 요청

---

## 1. Export: 현재 설정을 JSON으로 저장

```bash
# Realm 전체 Export (사용자 포함)
kc.sh export --dir /tmp/export --realm jtbc --users realm_file

# 사용자 제외 Export (Git 커밋용 권장)
kc.sh export --dir /tmp/export --realm jtbc --users skip
```

Export JSON에 포함되는 것:
- Realm 설정 (토큰 유효기간, 테마, 국제화 등)
- Clients (redirectUris, clientSecret 등)
- Roles (Realm Role, Client Role)
- Groups (Role 매핑 포함)
- Users (비밀번호 해시 포함 - Git에 올리면 안 됨)

## 2. 환경별 분리 전략

Export JSON을 그대로 다른 환경에 넣으면 URL이 개발 환경 것으로 고정됨.
`deploy.py`로 환경별 값을 치환 후 배포.

```python
# 환경별 설정
ENVIRONMENTS = {
    "dev":  {"app_base_url": "http://localhost:5001"},
    "stg":  {"app_base_url": "https://app-stg.jtbc.co.kr"},
    "prod": {"app_base_url": "https://app.jtbc.co.kr"},
}

# 사용법
python3 deploy.py --env dev       # 개발 환경 배포
python3 deploy.py --env stg       # 스테이징 배포
python3 deploy.py --env dev --dry-run  # 변환 결과만 확인
```

## 3. Import 방법 3가지

| 방법 | 시점 | 사용 케이스 |
|------|------|------------|
| `kc.sh import` | 서버 기동 전 | 초기 설치 |
| Admin REST API (PUT) | 런타임 | CI/CD 파이프라인 |
| Admin Console | 수동 | 긴급 복구 |

### Docker Compose 기동 시 자동 Import

```yaml
keycloak:
  command: start-dev --import-realm
  volumes:
    - ./jtbc-realm.json:/opt/keycloak/data/import/jtbc-realm.json
```

이미 Realm이 있으면 스킵. 없으면 자동 생성.

## 4. Git 관리 전략

```
keycloak-as-code/
├── jtbc-realm.json        ← Git 커밋 (users 제외 export)
├── deploy.py              ← 배포 스크립트
└── .gitignore
    ├── jtbc-realm-dev.json   # 환경별 파일 제외
    ├── jtbc-realm-stg.json
    └── jtbc-realm-prod.json
```

**주의사항**:
- `clientSecret`이 JSON에 포함됨 → Git Secret 또는 Vault로 관리
- 사용자 비밀번호 해시 포함 → `--users skip`으로 제외
- 운영 환경 JSON은 별도 private repo 또는 암호화 저장

## 5. CI/CD 파이프라인 예시

```yaml
# .github/workflows/keycloak-deploy.yml
on:
  push:
    paths: ['keycloak/jtbc-realm.json']

jobs:
  deploy:
    steps:
      - name: Deploy to Staging
        run: python3 deploy.py --env stg
        env:
          KEYCLOAK_ADMIN_PASSWORD: ${{ secrets.KC_STG_PASSWORD }}

      - name: Deploy to Production (main 브랜치만)
        if: github.ref == 'refs/heads/main'
        run: python3 deploy.py --env prod
```

## 6. 핵심 질문 & 답

**Q. Export JSON을 그대로 Git에 올리면 안 되는 이유 2가지는?**

> 1. `clientSecret`이 평문으로 포함됨 → 탈취 시 앱 사칭 가능
> 2. 사용자 비밀번호 해시가 포함됨 (`--users realm_file` 사용 시)
>    → 오프라인 해시 크래킹 공격 가능
>
> 해결: `--users skip`으로 사용자 제외, clientSecret은 Vault/Secret Manager로 관리
