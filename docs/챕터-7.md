# 챕터 7 (확장): 화이트라벨 테마

> 고객사 요청: 로그인 화면에서 "Keycloak" 문구 제거 및 자사 브랜딩 적용

---

## 1. 테마 시스템 구조

Keycloak 로그인 화면은 테마(Theme) 시스템으로 동작한다.
기본 테마를 복사하지 않고 **상속**으로 필요한 파일만 덮어쓴다.

```
/opt/keycloak/themes/
├── keycloak/          ← 기본 테마 (수정 금지)
└── jtbc/              ← 커스텀 테마 (Docker 볼륨으로 마운트)
    └── login/
        ├── theme.properties          ← parent=keycloak 선언
        ├── messages/
        │   └── messages_ko.properties ← 텍스트 덮어쓰기
        └── resources/
            └── css/
                └── login.css         ← 스타일 덮어쓰기
```

---

## 2. 파일별 역할

### theme.properties

```properties
parent=keycloak      # 기본 테마 상속
locales=ko,en        # 지원 언어
```

### messages_ko.properties

```properties
loginTitle=JTBC 통합 로그인
usernameOrEmail=아이디 또는 이메일
password=비밀번호
doLogIn=로그인
invalidUserMessage=아이디 또는 비밀번호가 올바르지 않습니다.
```

### login.css

```css
/* 배경, 버튼 색상, 로고 등 브랜딩 변경 */
#kc-login {
  background-color: #e8002d;  /* 회사 컬러 */
}
```

---

## 3. Docker Compose 마운트

```yaml
volumes:
  - ./themes/jtbc:/opt/keycloak/themes/jtbc
```

Keycloak은 `/opt/keycloak/themes/` 아래 디렉토리를 자동으로 테마로 인식한다.

---

## 4. 테마 적용

```bash
# kcadm.sh로 Realm에 테마 적용
kcadm.sh update realms/jtbc \
  -s loginTheme=jtbc \
  -s internationalizationEnabled=true \
  -s "supportedLocales=[\"ko\",\"en\"]" \
  -s defaultLocale=ko
```

---

## 5. 적용 결과 검증

```bash
# HTML에서 "Keycloak" 문자열 없음 확인
curl -s "http://localhost:8080/realms/jtbc/protocol/openid-connect/auth?client_id=account&response_type=code" \
  | grep -i keycloak
# → 결과 없음 (완전 제거)

# 커스텀 테마 적용 확인
# <title>JTBC 통합 로그인</title>
# <link href=".../jtbc/css/login.css" />
```

---

## 6. 변경 범위별 난이도

| 변경 범위 | 방법 | 난이도 | 업그레이드 영향 |
|---------|------|--------|--------------|
| 텍스트 변경 | messages_ko.properties | 쉬움 | 없음 |
| 색상/폰트 | login.css 덮어쓰기 | 쉬움 | 없음 |
| 레이아웃 변경 | login.ftl 덮어쓰기 | 중간 | 버전마다 병합 필요 |
| 완전 커스텀 UI | 새 .ftl 파일 작성 | 어려움 | 버전마다 병합 필요 |

**권장**: CSS + 메시지 파일만 덮어쓰는 방식.
`.ftl` 파일을 건드리면 Keycloak 업그레이드 시 템플릿 변경사항을 수동 병합해야 한다.

---

## 7. 로그인 화면 URL

```
http://<keycloak-host>/realms/<realm>/protocol/openid-connect/auth
  ?client_id=account
  &response_type=code
```
