# 챕터 9: LDAP / Active Directory 연동

> 가상 시나리오: 사내 AD 계정을 Keycloak에 또 만들어야 하냐는 질문

---

## 1. User Federation 개념

```
[이중 관리 - 나쁜 방식]
  AD: 김철수 계정 관리
  Keycloak: 김철수 계정 별도 관리 → 입퇴사 시 두 곳 수정 필요

[User Federation - 올바른 방식]
  AD: 김철수 계정 관리 (Single Source of Truth)
  Keycloak: AD에 인증 위임 → AD 비밀번호로 로그인
  AD에서 퇴사 처리 → Keycloak 로그인 즉시 차단
```

---

## 2. LDAP 핵심 용어

| 용어 | 의미 | AD 대응 |
|------|------|---------|
| DN (Distinguished Name) | 항목의 고유 경로 | 전체 경로 |
| dc (Domain Component) | 도메인 구성요소 | 도메인 |
| ou (Organizational Unit) | 조직 단위 (폴더) | OU |
| cn (Common Name) | 일반 이름 | 표시 이름 |
| uid | 사용자 ID | sAMAccountName |
| objectClass | 항목 유형 | 객체 클래스 |

**DN 예시**:
```
uid=lee.jiyeon,ou=people,dc=jtbc,dc=co,dc=kr
→ jtbc.co.kr 도메인의 people OU에 있는 lee.jiyeon
```

## 3. Keycloak LDAP Federation 설정

### kcadm.sh로 생성

```bash
kcadm.sh create components -r jtbc \
  -s name="jtbc-ldap" \
  -s providerId="ldap" \
  -s providerType="org.keycloak.storage.UserStorageProvider" \
  -s 'config.connectionUrl=["ldap://ldap-server:389"]' \
  -s 'config.usersDn=["ou=people,dc=jtbc,dc=co,dc=kr"]' \
  -s 'config.bindDn=["cn=admin,dc=jtbc,dc=co,dc=kr"]' \
  -s 'config.bindCredential=["admin_password"]' \
  -s 'config.usernameLDAPAttribute=["uid"]' \
  -s 'config.editMode=["READ_ONLY"]'
```

### 핵심 설정 파라미터

| 파라미터 | 의미 | AD에서 |
|---------|------|--------|
| `connectionUrl` | LDAP 서버 주소 | `ldap://ad-server:389` |
| `usersDn` | 사용자 검색 기준 DN | `CN=Users,DC=jtbc,DC=co,DC=kr` |
| `bindDn` | 조회용 서비스 계정 | `CN=svc-keycloak,CN=Users,...` |
| `usernameLDAPAttribute` | username으로 쓸 속성 | `sAMAccountName` |
| `editMode` | READ_ONLY / WRITABLE | 보통 READ_ONLY |

## 4. Group Mapper: LDAP 그룹 → Keycloak 그룹 동기화

```bash
# REST API로 Group Mapper 생성
curl -X POST http://keycloak/admin/realms/jtbc/components \
  -d '{
    "name": "ldap-group-mapper",
    "providerId": "group-ldap-mapper",
    "config": {
      "groups.dn": ["ou=groups,dc=jtbc,dc=co,dc=kr"],
      "group.name.ldap.attribute": ["cn"],
      "group.object.classes": ["groupOfNames"],
      "membership.ldap.attribute": ["member"],
      "mode": ["READ_ONLY"]
    }
  }'
```

동기화 후 결과:
```
LDAP: lee.jiyeon → 개발팀 그룹 멤버
         ↓ Group Mapper 동기화
Keycloak: lee.jiyeon → 개발팀 그룹 → ROLE_HR_ACCESS 상속
```

---

## 5. 동기화 방식

| 방식 | 설명 | 사용 케이스 |
|------|------|------------|
| 로그인 시 동기화 | 사용자가 로그인할 때 LDAP에서 최신 정보 가져옴 | 기본값, 권장 |
| 전체 동기화 | 모든 LDAP 사용자를 Keycloak DB에 미리 복사 | 검색 성능 필요 시 |
| 주기적 동기화 | 일정 간격으로 자동 동기화 | 대규모 조직 |

---

## 6. AD 연동 시 차이점

OpenLDAP과 Active Directory의 주요 차이:

| 항목 | OpenLDAP | Active Directory |
|------|---------|-----------------|
| username 속성 | `uid` | `sAMAccountName` |
| 그룹 objectClass | `groupOfNames` | `group` |
| 멤버 속성 | `member` | `member` |
| vendor 설정 | `other` | `ad` |
| 포트 | 389/636 | 389/636 |

AD 연동 시 `vendor=ad`로 설정하면 Keycloak이 AD 특화 동작을 자동 적용합니다.

```
lee.jiyeon (LDAP 사용자)

  권한: ROLE_HR_ACCESS 자동 상속
  토큰: realm_access.roles에 ROLE_HR_ACCESS 포함
```
